# domain/model

시스템의 **핵심 도메인 객체**가 위치하는 곳입니다.
비즈니스의 개념과 상태, 그리고 그 자체로 성립하는 규칙을 표현합니다.

## 무엇이 들어가는가
- **Entity**: 식별자(ID)를 가지고 생명주기를 갖는 객체
  - 예: `Dog`, `User`, `Diagnosis`
- **Value Object**: 식별자 없이 값으로 동등성을 판단하는 불변 객체
  - 예: `Weight`, `Breed`, `Email`
- **Enum / 도메인 상수**: 도메인 의미를 갖는 열거형
- 엔티티에 속한 불변식(invariant) 검증 로직

## 원칙
- 가장 안쪽 계층. **어떤 계층에도 의존하지 않는다.**
- 프레임워크(Spring, JPA 어노테이션 등)·외부 라이브러리에 의존하지 않는 순수 자바 객체(POJO)를 지향한다.
- 객체 스스로 유효한 상태만 갖도록 생성자/메서드에서 불변식을 지킨다.
